构建流程（5 阶段）

Phase 1 · 源码复制

src/ → build-src/，保持原始源码不变

Phase 2 · 源码转换

替换 feature('X') → false；替换 MACRO.X → 字符串字面值；移除 bun:bundle 导入

Phase 3 · 创建入口包装

生成 entry.ts，导入 entrypoints/cli.tsx

Phase 4 · 迭代打包（最多 5 轮）

esbuild 打包，自动处理缺失模块，生成 dist/cli.js (~12MB)

Phase 5 · 存根生成

自动创建缺失模块存根，支持文本资源和 JS/TS 模块

⚠️ 注意：项目原生基于 Bun 运行时（使用 bun:bundle、bun:ffi、feature() 宏等 Bun 特性），通过构建时转换适配 Node.js。108 个特性门控模块不在 npm 包中，存在功能缺失。
